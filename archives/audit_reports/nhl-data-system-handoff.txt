DATA SOURCE

API Endpoint: https://api-web.nhle.com/v1/standings/now
Authority: Official NHL API, verified against ESPN


FILES

JSON (single source of truth): nhl-standings-data.json (repo root)
Sync script: scripts/sync_nhl_data.py
GitHub Actions workflow: .github/workflows/daily-nhl-data.yml


AUTOMATION SCHEDULE

Runs daily at 11:00 AM ET (16:00 UTC)
Trigger: cron 0 16 * * * plus manual workflow_dispatch


VALIDATION CHECKS

1. Team count must equal 32
2. No null or missing values in any required field
3. Home W + Road W = Total W (every team)
4. Home L + Road L = Total L (every team)
5. Home OTL + Road OTL = Total OTL (every team)
6. Home GP + Away GP = Total GP (every team)
7. Pre-write validation runs before saving any files
8. Post-write validation re-reads JSON after all pages are written


FAILURE HANDLING

API fetch fails or returns fewer than 32 teams: exit code 1, no files touched
Pre-write validation fails: exit code 1, no files touched
Post-write validation fails: JSON restored from .bak backup, exit code 1
GitHub Actions: commit and push step only runs if sync script exits 0

KNOWN RISKS

1. ATS and O/U data is not automated. Action Network data on nhl-team-trends.html and nhl-totals-trends.html requires manual updates. The sync script only updates overall records and GF/GA on those pages.

2. NHL API schema changes. If NHL changes field names such as homeWins, the parser will fail gracefully (validation catches it), but the script will need a code fix.

3. Team name mapping. The script maps Utah Hockey Club to Utah Mammoth. If another franchise relocates or rebrands, TEAM_NAME_MAP in the script needs updating.

4. Season rollover. When the 2026-27 season starts, the season field in JSON and date patterns in HTML regex need updating.
